Luke Wingert Project: Map with HashTable

What is a map?

A map is a data structure in computer science that holds a series of ordered pairs. The notation of the pair is <(key, value)>. One practical use for an map might be prices of items on a menu at a restaurant. An example menu in map notatation is shown below. By looking up the key "burger" the code will get the value $6.00.

<(fries, $2.00), (burger, $6.00), (soda, $3.00)>

What is a hash table

For simplicity's sake think of number intervals. Say an interval was 1-10, 11-20 and another was 21-50. If you wanted to find the number 12, you would look in the interval 11-20 instead of 21-50. The intuition is similar for actual hash tables, Searching for a element in the data structure will be faster if the program knows which bucket to search in.

Combining a hash table and map

The idea is for each bucket in the hash table to contain it's own map. When the program needs to find a certain key, the hash function will tell the program which bucket to search in. This means in the best case scenario when all buckets are evenly filled with data, search time can decrease by (1/total buckets). Imagine searching through a fast food menu of 1 billion food items to find the price of fries!
